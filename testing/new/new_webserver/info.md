# Web Server

## How to Run

**1. Install dependencies** (only needed once):
```bash
pip install flask flask-cors
```

**2. Start the server:**
```bash
python webserver.py <port>
```

Example:
```bash
python webserver.py 8080
```

**3. Open the fake front end** in a browser:
```
http://localhost:8080/
```

The fake front end UI is served by the server itself at `/`.  
No separate server is needed to run the HTML page.

---

## File Overview

### `webserver.py`
The **entry point** of the back-end server. Responsibilities:
- Creates the Flask app and enables CORS
- Defines all HTTP route handlers (`/`, `/map`, `/timetable`, `/services/<id>`, `/tracking/<id>`, `/routes`)
- Validates and parses all incoming request parameters (type checking, missing fields, format checking)
- Calls the appropriate function in `webserver_task.py` to get the response body
- Uses `Server_Logger` to log every request, continuation, and response with event IDs
- Starts the Flask development server on the given port

Each route handler follows this pattern:
```
log_request → validate params → call task → log_response → return jsonify
```

---

### `webserver_task.py`
The **business logic layer** — contains the actual implementation of each API service.  
Each function takes validated, parsed Python types (not raw strings) and returns a `(dict, int)` tuple of `(response body, HTTP status code)`.

| Function | Endpoint | Fake Data | Description |
|---|---|---|---|
| `map_service(lat, long)` | `GET /map` | ✅ `sample_node.json` | Returns nodes for one of 3 pre-generated city areas (rotates per request), plus static weather metadata |
| `map_service_cached(lat, long)` | `GET /map?cached=true` | ❌ Stub | Returns an empty node list with static weather metadata only |
| `timetable_lookup(query_time, node_ids)` | `GET /timetable` | ✅ `sample_service_time_lookup.json` | Returns pre-generated timetable data for 3 stops with arrival times and service names |
| `service_lookup(node_id)` | `GET /services/<id>` | ✅ `sample_service_path.json` | Returns service routes for one of 3 pre-generated stops (rotates per request), with road geometry polylines |
| `live_tracking(node_id)` | ❌ Stub | `GET /tracking/<id>` | Always returns an empty vehicle list |
| `route_service(source_id, dest_id, query_time)` | `GET /routes` | ❌ Stub | Always returns an empty route list |

Also exposes `set_api_version(v)` and `set_server_logger(log)` so `webserver.py` can inject shared state.

**Fake data files** (in `fake_data/`):

| File | Generated by | Contents |
|---|---|---|
| `sample_node.json` | `generate_nodes()` | 3 arrays of `Node` objects (Lancaster, Preston, Manchester), each within a 1 km radius |
| `sample_service_time_lookup.json` | `generate_service_time_lookup()` | `TimetableLookup` response with arrival data for 3 NaPTAN stops |
| `sample_service_path.json` | `generate_service_path()` | 3 `ServiceLookup` responses, each listing bus lines and road geometry for one stop |

Regenerate fake data files by running `fake_data_generator.py` directly:
```bash
cd fake_data
python fake_data_generator.py
```

> `live_tracking` and `route_service` have no fake data yet. The `# TODO: populate from database` comments mark where database queries should be added.

### `webserver_component.py`
**Reusable server utilities** — uses Python standard library only (no external dependencies).

Contains the `Server_Logger` class which writes structured log entries to `server_log.log`.

| Method | Description |
|---|---|
| `log_init(port)` | Clears `server_log.log`, resets counter to 1, writes the `START` entry |
| `log_request(message)` | Logs an incoming request, returns an `event_id` integer |
| `log_continue(message, event_id)` | Logs a mid-request step, linked to its request by `event_id` |
| `log_response(message, event_id)` | Logs the outgoing response, linked to its request by `event_id` |

Log format:
```
[<counter>, <level>, <tag>, <timestamp>]   [<event type> : <event_id>] <message>
```

Example:
```
[       2,  INFO,   REQUEST,2026-02-26T22:56:04Z]   [Create Event   :       2] User request map service
[       3,  INFO,  CONTINUE,2026-02-26T22:56:04Z]   [Continue Event :       2] Server parse valid map arg
[       4,  INFO,  RESPONSE,2026-02-26T22:56:04Z]   [Finish Event   :       2] Server return map service
```

The `event_id` links all log lines belonging to the same request together, which is useful when multiple concurrent requests interleave in the log.

### `fake_front_end.html`
A **browser-based API tester UI** served at `GET /` by the Flask server.  
Provides one interactive card per API endpoint. Each card has input fields for the relevant parameters and a **Send Request** button that fires a real HTTP `fetch()` to the back end and displays the JSON response with syntax highlighting.

Does not require its own server — when opened via `http://localhost:<port>/`, all requests go to the same server automatically.

---

### `server_log.log`
Auto-generated log file created/cleared each time `webserver.py` starts.  
Written to by `Server_Logger`. Do not edit manually.

## External Dependencies

| Package | Purpose |
|---|---|
| `flask` | HTTP routing, request parsing, JSON responses |
| `flask-cors` | Adds `Access-Control-Allow-Origin` headers so the browser can call the API |

Install with:
```bash
pip install flask flask-cors
```
